## 2. 마일스톤

### M1 — AI 번역 버전 (현재 목표)
> 사용자가 직접 입력 → 캐릭터 선택 → AI 실시간 번역 → 결과 노출

### M2 — 음성 버전 (다음 목표)

```
사용자 입력 → AI API가 텍스트 번역 → Typecast API가 그 텍스트를 캐릭터 목소리로 재생
```

## 3. 캐릭터 정의

### 기본 캐릭터 (5종)

| 캐릭터 | 성격 | 번역 스타일 |
|------|------|---------|
| 키키 | 에너지 넘치고 감성적 | 공감으로 동기부여 |
| 부부 | 원칙주의자 | 데이터와 규칙으로 설득 |
| 차차 | 직설적 리더형 | 논리와 효율로 압박 |
| 나나 | 따뜻하고 감성적 | 마음에 호소 |
| 보보 | 현실적이고 유쾌함 | 직접적이지만 재치있게 |

### 캐릭터 × MBTI 구성 (16개 선택지)

- 5개 캐릭터가 각자 **다양한 모드(포즈/상황 이미지)** 보유
- 5개 캐릭터 × 다양한 모드 = **총 16개 선택지 (MBTI 16개 1:1 매칭)**
- 예: 운동하는 키키 → ENFP, 공부하는 키키 → ENFJ 이런 식으로 매핑
- 사용자는 16개 카드 중 1개 선택 → 해당 캐릭터 성격에 맞게 번역

## 4. User Flow

```
1. 페이지 접속
        ↓
2. 캐러셀 형태로 캐릭터 카드 노출
   - 상단: MBTI 텍스트 (예: ENTP)
   - 중앙: 캐릭터 이미지
   - 하단: 캐릭터 한줄 소개
   - 좌우 화살표(< >)로 넘기기
        ↓
3. 좌우 스와이프/화살표로 원하는 캐릭터 선택
   (넘길 때마다 MBTI + 이미지 + 소개 텍스트 변경)
        ↓
4. 아래 입력창에 하고 싶은 말 입력
   (예시 힌트 버튼: "청소 해!" / "공부 해!" / "연락 해!")
        ↓
5. 번역하기 버튼 클릭 → 로딩
        ↓
6. 선택한 캐릭터 성격에 맞게 번역된 결과 노출
        ↓
7. 앱 다운로드 CTA 노출
```

## 8. AI 프롬프트 구조

캐릭터별로 시스템 프롬프트를 다르게 넣어서 번역 스타일을 제어함
(사용할 AI API는 추후 확정)

```
[시스템 프롬프트 예시 - 키키 (ENFP)]
너는 키키야. ENFP 성격으로 에너지 넘치고 감성적이야.
사용자가 하고 싶은 말을 입력하면,
상대방이 기분 나쁘지 않으면서도 효과적으로 전달되도록
공감과 감성으로 번역해줘.
이모지를 적절히 활용하고, 따뜻하고 긍정적인 말투로 답해.

[유저 입력]
청소 해!

[번역 결과]
"야~ 방 치우고 나면 기분이 진짜 좋아질 거야!
같이 음악 틀고 치워보자 🎵✨"
```
